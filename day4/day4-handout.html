Tip: 설치가 안 될 때
  • "안 된다"를 감지: plugin list에 없다, unknown command
  • 의심 대상: marketplace list로 확인, plugin list --available
  • 검증 경로: @claude-code-guide 멘션, 공식 문서 URL 제공
  • 핵심: Claude에게 그냥 물어보면 잘못된 답 가능. 공식 문서 기반으로!

섹션 2

플러그인 만들기

내 스킬을 패키징 · 25분

소비자 → 생산자

Day 2에서 만든 스킬을
플러그인으로 묶어봅시다.

plugin.json 구조

{
  "name": "mrt-devkit",
  "description": "MyRealTrip development harness...",
  "version": "0.4.0",
  "author": { "name": "gildong.hong@myrealtrip.com" },
  "skills": [
    "./skills/bootstrap/SKILL.md",
    "./skills/setup/SKILL.md",
    "./skills/visualize/SKILL.md"
  ]
}

핵심 필드

name 플러그인 이름 (설치 시 사용)
description 설명
version 버전
skills 스킬 파일 경로 목록
agents 에이전트 파일 경로 목록

Step 1: 플러그인 생성

Claude에게 시키기

"Day 2에서 만든 [스킬 이름] 스킬을 플러그인으로 패키징해줘.
플러그인 이름은 'my-first-plugin'으로 하고,
~/my-first-plugin/ 디렉토리에 만들어줘."

생성 결과

~/my-first-plugin/
├── .claude-plugin/
│   └── plugin.json ← 메타데이터
└── skills/
    └── my-skill/
        └── SKILL.md ← Day 2에서 만든 스킬 
{
  "name": "my-first-plugin",
  "description": "나의 첫 플러그인",
  "version": "1.0.0",
  "author": { "name": "gildong.hong@myrealtrip.com" },
  "skills": [
    "./skills/my-skill/SKILL.md"
  ]
}

슬라이드 20의 mrt-devkit과 같은 구조. skills 경로가 실제 파일 위치와 일치해야 합니다.

Step 2: 로컬 테스트

claude --plugin-dir ~/my-first-plugin
  • /my-skill 슬래시 커맨드가 보이는지 확인
  • 실행하여 정상 동작 확인

Step 3: 검증

claude plugin validate ~/my-first-plugin

섹션 4

Agent SDK 데모

mari로 보는 확장 · 15분

Agent SDK란?

  • Claude Code의 핵심 엔진을 프로그래밍 가능한 SDK로
  • Skill, Agent, MCP를 그대로 사용 가능
  • 복사 붙여넣기로 다른 환경에 옮길 수 있음

API vs Agent SDK

같은 두뇌, 다른 몸

Anthropic API

  • Claude의 두뇌만 빌려오는 것
  • 대화 흐름, 도구 호출, 에러 처리를
    직접 만들어야 합니다
  • 비유: 엔진만 사서 자동차를 직접 조립
  • 💳 API Key 필수, 종량제 과금

Agent SDK

  • Claude Code의 완성된 몸을 가져오는 것
  • 대화 흐름, 도구, Skill, Agent 전부
    이미 만들어져 있습니다
  • 비유: 완성된 자동차를 가져다 색만 칠하기
  • ✓ Claude 구독료로 사용 가능

Claude Code = 50만 줄이 넘는 코드로 만들어진 완성품

Agent SDK를 쓰면, 이 50만 줄을 그대로 가져다 쓸 수 있습니다.

Agent SDK 코드

import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const message of query({
  prompt: "사용자의 요청",
