# 실습: 나만의 CLAUDE.md 만들기

> 이 템플릿을 참고하여 자신만의 CLAUDE.md를 작성해보세요.

---

## Step 1: 글로벌 CLAUDE.md (모든 프로젝트에 적용)

아래 내용을 참고하여 `~/.claude/CLAUDE.md`를 만드세요.

```bash
# 에디터로 열기 (nano 또는 원하는 에디터)
nano ~/.claude/CLAUDE.md
```

### 템플릿

아래에서 자신에게 맞는 항목을 골라 작성하세요:

```markdown
# 나의 선호 설정

## 언어
- 항상 한국어로 답변해주세요

## 스타일
- 존댓말을 사용해주세요
- 설명할 때 비유를 들어주세요
- 답변은 간결하게 해주세요

## 작업 방식
- 파일을 수정하기 전에 먼저 확인해주세요
- 한 번에 하나씩 작업해주세요
```

> 💡 위 항목은 예시입니다. 자신의 선호에 맞게 자유롭게 수정하세요!

---

## Step 2: 프로젝트 CLAUDE.md (이 프로젝트에만 적용)

프로젝트 폴더에서 `CLAUDE.md`를 만드세요.

```bash
cd my-first-project
nano CLAUDE.md
```

### 템플릿

```markdown
# 프로젝트 가이드

## 프로젝트 소개
이 프로젝트는 ___(프로젝트 설명)___ 입니다.

## 규칙
- 파일은 마크다운(.md) 형식으로 작성
- 파일명은 한국어 대신 영어로
- 제목은 반드시 포함할 것

## 폴더 구조
- docs/ : 문서 파일
- data/ : 데이터 파일
```

---

## Step 3: 확인하기

### CLAUDE.md가 제대로 로딩되는지 확인

```bash
# Claude Code 실행
claude

# 아래 명령어로 로딩된 파일 확인
/memory
```

### Claude에게 물어보기

```
이 프로젝트에 대해 설명해줘
```

```
나한테 어떻게 답변해야 하는지 알고 있어?
```

Claude가 CLAUDE.md의 내용을 이해하고 답하면 성공!

---

## Step 4: Git으로 기록하기

프로젝트 CLAUDE.md를 git에 저장하세요:

```bash
git add CLAUDE.md
git commit -m "프로젝트 CLAUDE.md 추가"
```

> 📌 `~/.claude/CLAUDE.md` (글로벌)은 개인 설정이므로 git에 올리지 않습니다.
> 프로젝트 `./CLAUDE.md`만 git에 커밋합니다.

---

## 보너스: 좋은 CLAUDE.md 작성 팁

1. **간결하게** — 200줄 이내를 목표로 하세요
2. **구체적으로** — "잘 작성해" ❌ → "마크다운 형식으로 제목 포함해서 작성해" ⭕
3. **구조적으로** — 마크다운 헤더와 불릿으로 정리하면 Claude가 더 잘 따릅니다
4. **모순 없이** — 서로 충돌하는 규칙이 있으면 Claude가 혼란스러워합니다
